Este projeto implementa o teste técnico de QA Engineer, cobrindo:
- critérios BDD em Gherkin
- automação de testes usando Cucumber + TypeScript
- testes manuais
- validação de três endpoints reais:
- sugestões de cidades
- previsão do tempo (7 dias)
- ranking de atividades baseado no clima
O objetivo é garantir clareza, cobertura e alinhamento com o comportamento real das APIs.
